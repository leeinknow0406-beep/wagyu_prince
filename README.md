# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
リモートリポジトリは,サーバー（Git Hub等）上にあるリポジトリで、<br>
ローカルリポジトリは,自分のPC上に作成されるリポジトリです。<br>
そのため、ローカルリポジトリをリモートリポジトリにPUSHする<br>
ことによりGit Hub等へ反映されるようになります。
## プッシュとマージの違いは何でしょうか？
プッシュは、ローカルレポジトリの変更をリモートリポジトリに反映させる操作のことですが、<br>
マージは、異なるブランチの変更を統合することです。
## コミットとプッシュの違い
コミットは、HTMLやCSSでの作業の変更点を保存する操作で、<br>
プッシュは、サーバー上のリポジトリに反映させる操作です。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
変更した内容を簡潔かつ明確に記述してあげるのが最適です。

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
主に、操作場所と実施するタイミングが違います。<br>
ローカルは、直接作業のブランチにマージさせるが、<br>
プルリクエストは、Git Hubのリポジトリ上で変更内容を他の開発者にレビューしてもらう<br>
という違いがあります。
## コンフリクトを起こしてしまった場合、どう対処すべきですか？
早めの対処法が必須。<br>
まず、コンフリクトしているファイルを特定する。<br>
次に、コンフリクトが発生している箇所を確認後、先または後、両方どれかのファイルを選択し、変更内容を取り込む。<br>
そして、再度ファイルをadd,commitのコマンドを使用し保存する。<br>
上記の方法で解消されると思います。
